# Weekly Documents

每个 `week-XX/` 目录对应一次阶段性迭代。

## 文件角色

- `prompt.md`：输入给 Claude Code 或其他 AI 编程工具的任务说明。尽量保留原始 prompt，便于复盘输入质量。
- `summary.md`：AI 编程工具执行后的过程总结，包括创建/修改了什么、为什么这样做、如何验证。
- `design.md`：可选。用于记录实现前或实现中的技术设计。
- `acceptance.md`：可选。用于记录验收标准、测试命令和最终结果。
- `notes.md`：可选。用于记录临时想法、排障过程、风险和 TODO。

## 周目录命名

使用两位数字：

```text
week-01/
week-02/
week-03/
```

这样在文件浏览器、GitHub 和命令行里都能自然排序。
